Functions/Methods in Java (return, parameters, method overloading)

public class Calculator {
    
    // Method 1
    public int add(int a, int b) {
        return a + b;
    }

    // Method 2
    public double add(double a, double b) {
        return a + b;
    }

    // Method 3 (âš  Problematic)
    public float add(float a, float b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();

        System.out.println(calc.add(5, 10));        // int version
        System.out.println(calc.add(5.5, 2.2));    // which version?
        System.out.println(calc.add(7.5f, 2.5f));  // float version
    }
}
//Why might the call calc.add(5.5, 2.2) not use the float version, even though it looks like floats?
//What is the method resolution rule (compile-time overloading + type promotion) that the JVM/compiler follows here?
//Fix the program so that all three overloaded methods are used correctly when called.
